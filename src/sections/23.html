¶ Возвращаемся к ходу компьютера

Допустим, текущая реализация истории нас устраивает. Мы можем вернуться к разработке хода компьютера.

Наш компьютер должен ходить в случайно выбранную клетку. Но как протестировать эту случайную клетку?

Мы можем изменить поведение Math.random в пределах теста, чтобы он возвращал одинаковое значение. Библиотека sinon в этом поможет. Также sinon позаботится, чтобы после завершения теста Math.random работал как обычно.

Подберём такое число, чтобы компьютер ходил в центр доски.

Тест падает, так как компьютер ходит в левую верхнюю клетку.

test/game.js
it('Computer sets it\'s move in randomly chosen cell', () => { 
  const stub = sinon.stub(Math, 'random').returns(0.5)
  const game = createGame()

  game.createComputerMove()
  const board = game.getState()

  expect(board[1][1]).to.equal(computerMoveSymbol)
  stub.restore()
})

Так как наше поле имеет размер 3×3, случайная координата должна быть ≥0 и ≤2. Пишем получение случайной координаты с использованием Math.random.

Новый тест проходит, но посыпались два других.

Первый упавший тест проверял, что компьютер должен сходить в левую верхнюю клетку. Этот тест устарел, поэтому его можно удалить.

Второй же тест проверяет, что игра записывает ход компьютера в историю. Здесь у нас есть два варианта действий. Мы можем так же заменить Math.random на фиктивный объект и тогда клетка будет всегда такой, которую мы укажем.

Либо мы можем проверять только поле turn, чтобы оно соответствовало мени компьютера.

Рефакторинг начнём с выноса повторяющегося кода.

Создадим метод для полчения случайной координаты _getRandomCoordinate. Размер поля вынесем в константу _fieldSize.

Запускаем тесты и проверяем.

+   this._fieldSize = 3
  
  // ...

-   const x = Math.floor(Math.random() * (3 - 0))
